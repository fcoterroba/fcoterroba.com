What is web scraping?

Web scraping consists of automatically browsing a website and extracting information from it. This can be very useful for many things and beneficial for almost any business. Today, I don‚Äôt think there‚Äôs a single successful company that doesn‚Äôt do it ‚Äîor doesn‚Äôt want to do it‚Äî.

The software programmed to scrape is usually called a bot, spider or crawler. Anyone can program a crawler, since there are tools to set it up that don‚Äôt require programming knowledge. That said, these tools will never give you all the flexibility you would have if you developed them in a programming language. Later we‚Äôll see the most used technologies and tools for creating these little critters.

Perhaps the typical captchas or the typical ‚ÄúI‚Äôm not a robot‚Äù that we normally encounter when filling out an online form have come to mind. Exactly! That‚Äôs what they protect themselves from crawlers. Because in addition to collecting information, they can also be used to fill out forms, create fake accounts or perform any action on the network automatically.

It‚Äôs true that there‚Äôs a legal and ethical debate about scraping. Later we‚Äôll see the legal risks it entails and how to carry it out responsibly.

It‚Äôs not so simple to say whether scraping is legal or not. It depends a lot on each case and, even so, most of the time it won‚Äôt be very clear. That‚Äôs why, although it‚Äôs unlikely, you always have to assume the risk before scraping that the scraped company can file a complaint. Although the most likely thing is that, in case of taking some response action, it would be to block the bot or give some kind of warning.

That said, before starting to scrape, it‚Äôs more than recommended to check if an API is provided to access the data without having to scrape. For example, Idealista, the well-known rental and home buying website, offers an API so both sides win. On the one hand, making requests to an API requires less development than scraping and, on the other, Idealista avoids all that ‚Äúundesired‚Äù traffic. Also, by opening their data to the public, they give someone the opportunity to develop something powerful for them.

If you haven‚Äôt had luck and they don‚Äôt provide an API, you‚Äôll have to read the terms and conditions, to check that they don‚Äôt say anything regarding the automatic extraction of information. Anyway, before starting to scrape you‚Äôll have to consider if the number of requests needed are too abusive and if the purpose of the project harms the website‚Äôs business. There could be more objections from the scraped company if you‚Äôre profiting from their content, since you can be sued for violation of intellectual property rights. Especially if the content you want to extract is behind a user login, since, in this case, we‚Äôre definitely not talking about public domain information.

In conclusion, in order to avoid scares, it‚Äôs recommended to contact the company to tell them what the project consists of or, even, reach some agreement with them. Also, it‚Äôs always a good idea to get advice from an expert lawyer.
